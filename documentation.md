# Документация для главной страницы интернет-магазина AppleStore

## 1. Общая информация

### 1.1. Назначение проекта
Главная страница интернет-магазина AppleStore предназначена для представления продукции Apple целевой аудитории с высоким уровнем дохода (от 150 000 рублей в месяц) в возрасте 20-35 лет. Страница выполняет функции навигационного хаба, знакомит пользователей с основными категориями товаров и предоставляет контактную информацию.

### 1.2. Основные функции
- Представление бренда AppleStore как официального дилера
- Навигация по основным категориям товаров (iPhone, iPad, MacBook)
- Предоставление контактных данных магазина
- Адаптивное отображение на всех типах устройств
- Минималистичный пользовательский интерфейс в стиле Apple

## 2. Архитектура проекта

## 3. Компоненты интерфейса

### 3.1. Шапка (Header)
**Расположение:** `header` с классом `.header`
**Фиксированная позиция:** Да (с тенью для отделения от контента)
**Состав:**
- Логотип AppleStore (SVG + текст) с ссылкой на главную
- Иконка пользователя (Font Awesome) с ссылкой на личный кабинет

**Поведение:**
- При скролле > 50px уменьшается высота и размер логотипа
- При наведении на иконку пользователя - плавное увеличение и изменение цвета

### 3.2. Герой-секция (Hero Section)
**Расположение:** `section` с классом `.hero`
**Особенности:**
- Полноразмерное фоновое изображение с полупрозрачным затемнением
- Крупный заголовок H1
- Подзаголовок с описанием
- Кнопка призыва к действию "Смотреть каталог"

**Анимации:**
- Плавное появление при загрузке
- Эффект поднятия кнопки при наведении

### 3.3. Сетка категорий (Categories Grid)
**Расположение:** `section` с классом `.categories`
**Структура:** 3 карточки в строку (на десктопе) или в колонку (на мобильных)

**Карточка категории включает:**
- Фоновое изображение категории
- Полупрозрачное затемнение для читаемости текста
- Название категории
- Кнопку "Смотреть все"

**Интерактивность:**
- При наведении: поднятие карточки, усиление тени, изменение цвета кнопки
- Вся карточка кликабельна (ведет в соответствующий раздел каталога)
- Анимация появления карточек при скролле

### 3.4. Подвал (Footer)
**Расположение:** `footer` с классом `.footer`
**Структура (3 колонки на десктопе):**
1. **О магазине:** логотип и краткое описание
2. **Контакты:** телефон, email, адрес (ссылки `tel:` и `mailto:`)
3. **Быстрые ссылки:** навигация по основным категориям

**Нижняя часть подвала:**
- Разделительная линия
- Копирайт
- Ссылки на юридические документы

## 4. Технические характеристики

### 4.1. Цветовая схема
| Элемент | HEX | Назначение |
|---------|-----|------------|
| Основной акцент | #308582 | Фон шапки и подвала |
| Фон контента | #ffffff | Основная область |
| Основной текст | #1d2029 | Текст на светлом фоне |
| Текст ховер | #287370 | Эффект наведения на ссылки/кнопки |
| Текст в шапке/подвале | #ffffff | Текст на темном фоне |
| Вторичные элементы | #8a8f9c | Неактивные элементы |
| Разделители | #eaeaea | Границы между элементами |

### 4.2. Типографика
**Стек шрифтов:**
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 
             'Helvetica Neue', Arial, sans-serif;
```

**Размеры шрифтов:**
- Hero заголовок: 3rem (десктоп) → 1.8rem (мобильный)
- Заголовки категорий: 2.2rem → 1.8rem
- Основной текст: 1rem
- Мелкий текст: 0.9rem

### 4.3. Адаптивность
**Брейкпоинты:**
- Десктоп: 1025px+
- Планшет: 769px - 1024px
- Мобильный: 320px - 768px

**Адаптивные изменения:**
- Шапка: изменение отступов и размеров
- Герой-секция: изменение размеров текста и отступов
- Сетка категорий: 3 → 2 → 1 колонка
- Подвал: 3 → 2 → 1 колонка

### 4.4. Анимации и переходы
**Базовые настройки:**
```css
transition: all 0.3s ease;
```

**Типы анимаций:**
1. **Ховер-эффекты:** изменение цвета, масштаба, положения
2. **Появление элементов:** плавное проявление при скролле
3. **Эффект поднятия:** для карточек категорий
4. **Плавный скролл:** к якорным ссылкам

## 5. Производительность и оптимизация

### 5.1. Изображения
**Подход:** 
- Использование CSS-градиентов поверх изображений для улучшения читаемости
- Фоновые изображения через CSS (оптимизированы для Retina-дисплеев)
- В продакшене рекомендуется:
  - Конвертация в WebP/AVIF с fallback
  - Использование атрибута `srcset` для адаптивных изображений
  - Ленивая загрузка (lazy loading)

### 5.2. Кодовая база
**Оптимизации:**
- Минимизированные CSS-правила (использование CSS-переменных могло бы улучшить)
- Эффективное использование Flexbox и Grid
- Минимальное количество JavaScript (всего 2.5KB несжатого кода)
- Семантическая верстка для лучшей SEO-оптимизации

### 5.3. Загрузка ресурсов
**Внешние зависимости:**
- Font Awesome: загружается асинхронно через CDN
- Google Fonts: подключен, но не используется напрямую (системные шрифты имеют приоритет)
- Изображения Unsplash: в демо-версии, в продакшене должны быть локальными

## 6. Кроссбраузерная совместимость

### 6.1. Поддерживаемые браузеры
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 6.2. Особенности реализации
- CSS Grid с фолбэком на Flexbox для старых браузеров
- Префиксы vendor не требуются для указанных версий браузеров
- Progressive Enhancement: базовая функциональность доступна даже без JavaScript

## 7. Доступность (Accessibility)

### 7.1. ARIA-атрибуты
- Иконка пользователя: `aria-label="Личный кабинет"`
- Все ссылки имеют понятные тексты
- Логичный порядок фокуса (Tab order)

### 7.2. Семантическая верстка
- Использование тегов `<header>`, `<main>`, `<section>`, `<footer>`
- Правильная иерархия заголовков (H1 → H2 → H3)
- Альтернативные тексты для изображений (в реальном проекте через `alt`)

### 7.3. Контрастность
- Соответствие WCAG AA: контраст текста на фонах превышает 4.5:1
- Цветовые переходы при наведении не нарушают читаемость
